# 🧠 First Principles Understanding

## Why Do Databases Exist?

Computers lose memory when applications stop.

Example:

```js
const users = [];
```

When the server restarts:

```text
users = []
```

Everything disappears.

---

## First Principle

A database solves one problem:

> Store information reliably and retrieve it efficiently.

---

## Data Flow

```text
Client
   ↓

   ↓
Database
   ↓
Persistent Storage
```

# 2️⃣ SQL vs NoSQL

## First Principles Comparison

### SQL

Stores data in tables.

```text
Users Table

ID | Name
---------
1  | John
2  | Sarah
```

---

### NoSQL

Stores data as documents.

```json
{
  "_id": "123",
  "name": "John"
}
```

---

# SQL Characteristics

* Structured schema
* Tables
* Relationships
* Strong consistency

Examples:

* PostgreSQL
* MySQL
* SQL Server

---

# NoSQL Characteristics

* Flexible schema
* Document storage
* Horizontal scalability
* Fast iteration

Examples:

* MongoDB
* CouchDB
* DynamoDB

---

# Comparison

| Feature        | SQL       | NoSQL      |
| -------------- | --------- | ---------- |
| Structure      | Tables    | Documents  |
| Schema         | Fixed     | Flexible   |
| Relationships  | Strong    | Weak       |
| Scaling        | Vertical  | Horizontal |
| Transactions   | Excellent | Limited    |
| Learning Curve | Moderate  | Easy       |

---

## Checklist

* [ ] Explain SQL databases.
* [ ] Explain NoSQL databases.
* [ ] Understand tradeoffs.
* [ ] Choose appropriate database type.

---

## Veteran Insight

There is no "best database."

Choose based on:

* Data structure
* Scaling needs
* Consistency requirements

# 3️⃣ Introduction to MongoDB

## What is MongoDB?

MongoDB is a NoSQL document database.

Instead of tables:

```text
Database
   ↓
Collection
   ↓
Document
```

---

## Example

Collection:

```text
users
```

Document:

```json
{
  "_id": "1",
  "name": "John",
  "email": "john@example.com"
}
```

---

## MongoDB Concepts

| SQL      | MongoDB    |
| -------- | ---------- |
| Database | Database   |
| Table    | Collection |
| Row      | Document   |
| Column   | Field      |

---

* [ ] Insert documents.

# 4️⃣ Connecting MongoDB to Node.js

## Why Mongoose?

MongoDB driver is low-level.

Mongoose provides:

* Schema validation
* Models
* Middleware
* Cleaner syntax

---

## Installation

```bash
npm install mongoose
```

---

## Connection Example

```js
const mongoose = require("mongoose");

mongoose.connect(
    process.env.MONGO_URI
);
```

---

## Connection Flow

```text
Node.js App
      ↓
Mongoose
      ↓
MongoDB Driver
      ↓
MongoDB Database
```

---

## Checklist

* [ ] Handle connection errors.
* [ ] Use environment variables.

# 5️⃣ Mongoose Schemas and Models

## Why Schemas Exist

Without schemas:

```json
{
  "name": "John"
}
```

and

```json
{
  "abc": true
}
```

could coexist.

Chaos.

---

## Schema Example

```js
const userSchema =
new mongoose.Schema({
    name: String,
    email: String
});
```

---

## Model Example

```js
const User =
mongoose.model(
    "User",
    userSchema
);
```

---

# 9️⃣ ORMs (Object Relational Mappers)

## Why ORMs Exist

Without ORM:

```js
const result =
await db.query(
"SELECT * FROM users"
);
```

---

With ORM:

```js
const users =
await User.findMany();
```

Cleaner and safer.

---

## Benefits

* Productivity
* Type Safety
* Migrations
* Validation

---

# Prisma

## Installation

```bash
npm install prisma
```

---

## Strengths

* Excellent developer experience
* Type-safe queries
* Modern ecosystem

---

## Example

```js
await prisma.user.findMany();
```

---

# Drizzle

## Why Drizzle?

Modern lightweight ORM.

---

## Strengths

* SQL-first
* Excellent TypeScript support
* Better performance

---

## Example

```js
await db.select()
.from(users);
```

---

## ORM Comparison

| ORM       | Strength             |
| --------- | -------------------- |
| Prisma    | Developer Experience |
| Drizzle   | SQL-first Approach   |
| Sequelize | Legacy Projects      |
| TypeORM   | Enterprise Apps      |

---

## Checklist
