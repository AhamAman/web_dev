# AWS EC2: Virtual Servers in the Cloud (Beginner → Veteran Roadmap)

## Learning Objective

Master Amazon EC2 from first principles, understanding not only how to launch instances but why virtual servers exist, how cloud infrastructure works underneath, and how to architect scalable, secure, and production-ready compute systems.

---

# Phase 0: First-Principles Foundation

## Core Question

Before learning EC2, answer:

* [ ] What is a server?
* [ ] Why do applications need servers?
* [ ] Why not run applications on personal laptops?
* [ ] What problems do physical servers create?
* [ ] Why did virtualization become necessary?

### First-Principles Model

Application → Operating System → Hardware

Traditional Infrastructure:

Application → OS → Physical Server

Cloud Infrastructure:

Application → OS → Virtual Machine → Hypervisor → Physical Server

### Key Insights

* [ ] A server is simply a computer serving requests.
* [ ] EC2 is a rented virtual computer.
* [ ] AWS manages hardware so you manage software.

### Mastery Check

* [ ] Explain EC2 without using AWS terminology.
